# Project Planning

## Goal

Break work into phases, backlog items, acceptance criteria, and risks.

## When To Use

- Planning a new feature
- Re-scoping a release
- Turning a large request into implementation steps

## Steps

1. Define the user-facing outcome.
2. Separate packaging, domain, application, and adapter work.
3. Identify public CLI or API changes.
4. Write acceptance criteria and risks.
5. Propose small commits in dependency order.

## Checklist

- Backlog is ordered
- Acceptance criteria are testable
- Risks have mitigations
- Architectural boundaries are respected

## Exit Criteria

A developer can start implementation without guessing scope or order.
