# Report Template Design

## Goal

Create prompt templates that reliably yield structured Markdown reports.

## When To Use

- Adding a new report template
- Revising prompt instructions
- Aligning report structure across providers

## Steps

1. Define the intended output audience and format.
2. Keep headings and required sections explicit.
3. Preserve traceability by storing the rendered prompt.
4. Keep templates packaged as installable resources.
5. Add coverage for discovery and loading.

## Checklist

- Template has a clear use case
- Output structure is explicit
- Template is listed by `templates list`

## Exit Criteria

The template can be selected, rendered, and traced after installation.
