# Product Context

## Problem

Users often have recordings of meetings, interviews, classes, or technical sessions but need a repeatable way to turn those recordings into traceable written reports.

## Target Users

- Individual professionals who process recorded meetings
- Researchers and journalists handling interviews
- Students converting lectures into notes
- Technical teams generating structured summaries from demos or incident reviews

## Primary Use Cases

- Process a single audio or video file
- Process a directory of recordings
- Keep all intermediate artifacts next to the source file
- Use local-first defaults with optional remote LLM generation

## Product Boundaries

- CLI only, no GUI
- Installable from PyPI
- Linux and macOS first
- Bootstrap release focuses on package structure, config, command surface, and artifact planning
